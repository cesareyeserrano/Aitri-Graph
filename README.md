# Aitri Graph Visualizer

Interactive dependency graph for software artifacts generated by [Aitri](https://github.com/cesareyeserrano/aitri). Visualize Requirements, User Stories, and Test Cases as a navigable, collapsible graph — loaded from a GitHub repository or a local Aitri project path.

![Node.js](https://img.shields.io/badge/Node.js-v18%2B-green) ![Tests](https://img.shields.io/badge/tests-40%2F40-brightgreen) ![License](https://img.shields.io/badge/license-ISC-blue)

---

## Features

- **Load from GitHub** — paste a public GitHub repo URL and the graph loads automatically
- **Load from local path** — point to any local Aitri project directory
- **Hierarchical layout** — Requirements → User Stories → Test Cases rendered with Cytoscape.js + dagre
- **Expand / collapse subtrees** — click any node to collapse its descendants
- **Dependency edges** — dashed directed edges for cross-node dependencies
- **Status indicators** — color-coded per node status (pending, in_progress, approved, complete, drift)
- **Tooltips** — hover any node to see id, title, type, status, and dependencies
- **Persistent project registry** — projects survive page reloads and browser clears
- **Zoom / pan / fit** — mouse wheel zoom, drag to pan, fit-to-screen button
- **Zero build step** — one command to start

## Quick start

```bash
node server.js
```

Open [http://localhost:3000](http://localhost:3000).

> Requires Node.js v18+. No `npm install` needed — the server uses only Node.js built-ins.

Override port:

```bash
PORT=8080 node server.js
```

## Usage

### Add a GitHub project

Paste a public GitHub repo URL that contains an Aitri `spec/` directory:

```
https://github.com/owner/repo
```

### Add a local project

Paste the absolute path to any local Aitri project:

```
/Users/you/projects/my-aitri-project
```

The project must contain `spec/01_REQUIREMENTS.json`. `spec/03_TEST_CASES.json` and `.aitri` are optional.

### Switch between projects

Click any project in the sidebar to switch the graph. Projects persist across reloads.

### Remove a project

Hover the project item in the sidebar and click the × button.

## API

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/project?path=<abs>` | Read Aitri artifacts from a local path |
| `GET` | `/api/project/status?path=<abs>` | Lightweight status poll (phase, updatedAt) |
| `GET` | `/api/registry` | List persisted projects |
| `POST` | `/api/registry` | Register a project |
| `PATCH` | `/api/registry` | Update active project |
| `DELETE` | `/api/registry?url=<encoded>` | Remove a project |

Security: paths containing `..` or relative paths return HTTP 400.
